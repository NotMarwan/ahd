# تصميم: تجربة المستخدم والواجهة — «يصوَّر كمنتج فينتك سعودي مشحون» · 16 يوليو 2026

**توجيه المالك (16 يوليو):** التركيز القادم كله على تصميم التطبيق والواجهة وتجربة المستخدم.
**البوصلة:** بند «تجربة المستخدم» في `docs/JUDGE-LENS.md` — «كل شاشة يصلها المحكّم تُصوَّر
كمنتج فينتك سعودي مشحون؛ عربي أولًا، حركة توضّح، صفر زوايا غير منسّقة، ومسار العرض
مُخرَج إخراجًا لا نقرًا عشوائيًا». الأساس المرجعي: `docs/DESIGN.md` (النظام)،
`docs/AHD-FULL-DESIGN.md` (بنية المعلومات والتجاوب)، لوحات الثيم في ملفات benchmark-ar
للمراجع السبعة، وتوجيه «عصري معتدل» المعتمد في موجة القدرات (3380/0).

## 1. التشخيص (لماذا هذه الخطة الآن)

- القدرات صارت منافسة (موجة 16 يوليو)، لكن كثافة الشاشات تتفاوت: شاشات وُلدت في موجات
  مختلفة تحمل إيقاعات هوامش وأحجام عناوين متفاوتة.
- الحركة شبه غائبة: لا انتقالات توضّح السبب والنتيجة (ختم، قبول، مقاصّة) — بند الحركة
  في عدسة التحكيم مكشوف.
- الجوّال (عرض ≤ 480px) يعمل لكنه «مضغوط» لا «مصمَّم»: الـ nav صف تمرير أفقي، والبطاقات
  عمود واحد بلا ترتيب أولويات.
- قرارات المراجع البصرية لم تُستثمر بعد: كل benchmark فيه «الأنسب لعهد» بصري لم يُنقل.

## 2. الاتجاه المعتمد (قرارات، لا خيارات)

1. **سادو تبقى الهوية، والڤايب عصري:** لا استبدال للنظام (`--sadu-*` tokens)؛ نرفع
   العصرية بالإيقاع والحركة والتباين — لا بتغيير الألوان أو الخط.
2. **درس كل مرجع بصريًا (من لوحات الثيم):**
   - Zirtue → وضوح الخطوة والمبلغ والجدول (شريطنا «وش الوضع؟» يتصدّر بصريًا).
   - Splitwise → ترميز «لك/عليك» لونيًا ثابتًا في كل الشاشات + عدم الاعتماد على اللون وحده.
   - Hakbah → دفء محلي ورسائل نجاح قصيرة إنسانية؛ خلفيات نظيفة.
   - MoneyFellows → طاقة الهدف والتقدم (أرقام hero كبيرة) مع تخفيف لغة البيع.
   - ناجز → انضباط وطمأنينة في لغة الإثبات والمرجع، دون شكل حكومي.
   - DocuSign → فعلٌ أساسي واحد بارز لكل شاشة؛ إطار المستند بطلًا في لحظة الختم.
3. **الحركة توضّح ولا تزيّن:** transitions/animations بـ CSS فقط (transform/opacity)،
   حتمية (لا JS timers ولا random)، مع `prefers-reduced-motion` كاملًا. لحظات الحركة
   الأربع: الختم، القبول/التصديق، انهيار المقاصّة، تمزّق العبث.
4. **الجوّال مواطن درجة أولى:** تفعيل غلاف `docs/AHD-FULL-DESIGN.md §7` — هاتف ≤480:
   شريط سفلي بخمس وجهات (الرئيسية، أنشئ، دفتري، السجل، المزيد) + بطاقات كاملة العرض؛
   لوحي 481–1024: شبكة عمودين؛ سطح المكتب: كما هو مع حد أقصى للعرض.
5. **الحدود غير القابلة للمساومة:** الأحمر للعبث فقط؛ التأخر كهرمان دائمًا؛ لا مؤقتات
   عدّ؛ لا ثقة رقمية؛ الخط IBM Plex Sans Arabic vendored فقط (لا CDN)؛ حتمية البايتات
   في كل منطق؛ حد literals الخط في app.css ≤ 200 (بوابة sadu-tokens).

## 3. النطاق

**داخل:** app/app.css + app/styles/sadu-tokens.css (إضافات)، تحسينات HTML طفيفة في
الشاشات (ترتيب/أصناف)، شاشات مسار العرض أولًا (home → create → proof → settle → jamiya
→ daftari/mine)، ثم البقية. اختبارات smoke تُحدَّث تقويةً لا إضعافًا.
**خارج:** الديمو المجمّد، الدوال الذهبية، تطبيق Expo الجوال (موجة لاحقة)، تغيير الشعار
أو الألوان الأساسية، أي مكتبة خارجية.

## 4. معيار الإنجاز

- لقطات 320/768/1024/1440 لكل شاشة من مسار العرض: صفر overflow أفقي، صفر زوايا غير منسّقة.
- مراجعة judge-panel (إجراء JUDGE-LENS الست وكلاء): بند UX ≥ 8 والتذكّر ≥ 8، وإلا JL-.
- `cd tests && node run-all.cjs` أخضر بعد كل مرحلة؛ tripwire سليم.
- التنفيذ على فرع `claude/design-ux-*` — لا دمج بـ main إلا بمراجعة المالك.
