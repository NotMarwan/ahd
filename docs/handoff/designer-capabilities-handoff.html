⚠️ هذه الصفحة ليست مرجعية تصميم. الشكل الحالي في التطبيق مؤقت (placeholder) — التصميم البصري كله لك. المطلوب هنا: افهم القدرات وسلوكها وتدفقاتها، والتزم فقط بالثوابت الدلالية الست في آخر الصفحة. لوحات الثيم أدناه لفهم «لماذا وُجدت كل قدرة» عند المنافسين — لا لنسخ شكل أحد.

أولًا: شغّل التطبيق عندك

  1. اسحب المستودع: git clone https://github.com/NotMarwan/ahd.git (فرع main).
  2. شغّل الخادم المحلي: node app/_serve-app.cjs — يحتاج Node ‏18+ فقط، بلا تثبيت حزم، ويعمل دون إنترنت.
  3. افتح http://localhost:8124 — وامشِ على «جرّبها» في كل قدرة أدناه بنفس الترتيب.

المرجع الهيكلي الكامل (تواقيع الدوال، الحالات، الأفعال): docs/CAPABILITIES-STRUCTURE.md في المستودع.

ثانيًا: القدرات التسع — وكيف تجرب كل واحدة

١

شريط «وش الوضع؟»

المرجع: Zirtue + ناجز

كل عهد يجيب ثلاثة أسئلة: المتفق عليه؟ ما حدث؟ ما التالي؟ — مع رقم مرجعي بصيغة عهد-XXXX ونبرة انتباه كهرمانية (ليست حمراء أبدًا).

جرّبها: افتح الرئيسية — البطاقة تظهر فورًا لأكثر عهد يحتاج انتباهك. ثم افتح «دفتري» و«ما عليّ»: كل صف يحمل سطر «التالي» ومرجعه.

مقابض CSS الجاهزة لإعادة التشكيل: .nsx و .nsx-ref و .row-next

٢

مراجعة قبل الختم

المرجع: ناجز + DocuSign

لا يُختم شيء مباشرة: زر «راجع قبل الختم» يفتح ملخصًا ثابتًا (الأطراف، المبلغ، الجدول) + قائمة «ما ليس في هذا العهد» (لا فائدة، لا غرامة، لا حيازة) + بصمة معاينة، ثم «أكّد واختم».

جرّبها: «أنشئ عهدًا» → راجع قبل الختم → أكّد واختم عبر نفاذ (محاكاة). نفس الخطوة في «اطلب عهدًا» قبل الإرسال.

مقابض CSS الجاهزة لإعادة التشكيل: .rv-card و .rv-abs و .rv-fp

٣

تصديق السداد

المرجع: ناجز

المدين يسجّل دفعة بمؤيد وصفي (حوالة، نقدًا أمام شاهد…) — الرصيد لا يتغيّر. الدائن يصدّق (عندها فقط يُختم السداد وينقص المتبقي) أو يرفض بسبب من قائمة ثابتة، والرفض يفتح «محلّ خلاف» والسجلان محفوظان.

جرّبها: «ما عليّ» → «سجّلت دفعة — اطلب تصديقها» → أرسل → لاحظ شارة الانتظار وصف المحاكاة (صدّق/ارفض). ثم «دفتري» يعرض «دفعات بانتظار تصديقك».

مقابض CSS الجاهزة لإعادة التشكيل: .pc-box و .pc-claim و .pc-pending و .pc-form

٤

أنماط التقسيم

المرجع: Splitwise

قسمة الفاتورة بأربعة أنماط: متساوٍ، مبالغ محددة، نسب مئوية، حصص — وكلها تحفظ كل هللة (أكبر باقٍ للأسبق). المعاينة قبل الحفظ إلزامية، والمجموع الخاطئ يعطّل زر التطبيق برسالة عربية.

جرّبها: «اليومي» → قسمة فاتورة → غيّر «طريقة التقسيم» → «عاين الحصص قبل الحفظ» → طبّق. جرّب مجموعًا خاطئًا وشاهد الرسالة.

مقابض CSS الجاهزة لإعادة التشكيل: #daily-split-mode و #daily-split-values

٥

مرشح الشخص

المرجع: Splitwise

دفتري يُرشَّح بالشخص إضافة إلى الحالة — الرقاقتان تتركبان معًا.

جرّبها: «دفتري» → صف رقائق الأشخاص فوق القائمة (الكل، سلطان، مقهى الحي…).

مقابض CSS الجاهزة لإعادة التشكيل: .dperson

٦

دعوات الجمعية

المرجع: هكبة

بطاقة دعوة لكل عضو تعرض كل الشروط (المساهمة، المدة، دوره بالتحديد) + «ما ليس في هذه الجمعية» (لا رسوم، لا حيازة، لا سند تنفيذي). الختم مقفول حتى يقبل الجميع قبولًا فرديًا مسجلًا — واعتذار واحد يعيد قفله.

جرّبها: «الجمعية» → بطاقة الدعوة → اضغط «يقبل (محاكاة)» للستة وشاهد زر الختم ينفتح؛ ثم «يعتذر» لأي عضو وشاهده يُقفل.

مقابض CSS الجاهزة لإعادة التشكيل: .ji-box و .ji-row

٧

هدف الجمعية وسجل تغييراتها

المرجع: MoneyFellows + هكبة

الجمعية تبدأ بهدف وصفي وشريط تقدم مع سطر ثابت «هدف وصفي — لا وعد مالي ولا عائد»، وجدول مقارنة سيناريوهات قبل الدعوة. وكل تبديل دورين بالتراضي أو انسحاب = قيد مرقّم في سجل إلحاقي لا يُمحى، وتُختم نسخة جديدة من العقد دون فقد أي دفعة.

جرّبها: نفس الشاشة: قسم الهدف أعلى، وجرّب «بدّل آخر دورين بالتراضي (محاكاة)» أسفل وشاهد القيد JC-1 يظهر.

مقابض CSS الجاهزة لإعادة التشكيل: .jam-goal و .jam-scen و .jam-changes

٨

المتكرر مسودات

المرجع: Splitwise

القسمة الشهرية المتكررة لا تنشر نفسها: كل دورة تُقترح مسودةً تنتظر «اعتمد وانشر» أو «أهمل» بسبب محفوظ. لا التزام آليًا أبدًا.

جرّبها: «الدائرة+» → قسم «قِسْمة دائمة · الإيجار» → اعتمد الدورة الأولى وأهمل الثانية وشاهد الحالتين.

مقابض CSS الجاهزة لإعادة التشكيل: شارات «مسودة / نُشرت / أُهملت» داخل .ca-line

٩

موافقات المقاصّة

المرجع: الجميع

كل حوالة دنيا (رِجل) بطاقة تحتاج موافقة طرفيها، مع سطر أثرها على كل طرف. «اعتماد المقاصّة» مقفول حتى تكتمل كل الموافقات، وتغيير المثال يبدأ جولة موافقات جديدة. برهان حفظ المراكز يبقى معروضًا.

جرّبها: «المقاصّة» → وافق لطرفي كل حوالة → شاهد «الرِّجل جاهزة» ثم زر الاعتماد ينفتح.

مقابض CSS الجاهزة لإعادة التشكيل: .se-consent و .se-consent-gate و .se-approved

ثالثًا: لوحات الثيم — المراجع الستة (للفهم لا للنسخ)

لقطات رسمية عامة لكل مرجع + قراءتها البصرية كما وثّقها فريق البحث. سطر «لماذا يهمنا» هو الخلاصة الوحيدة المقصودة.

رابعًا: الثوابت الدلالية الست (حدودك الوحيدة)

  • الأحمر محجوز لكشف العبث المشفّر فقط (كسر الختم). التأخر في السداد كهرمان دائمًا — مهما بلغ.
  • لا عدّادات أيام، لا أرقام ثقة، لا نسب مئوية على شخص. إشارة الثقة كلمة: «وفّى بعهوده».
  • الرصيد لا يتغيّر صامتًا أبدًا — كل تغيير بفعل صريح من صاحب الحق.
  • الرفض دائمًا مسبب ومحفوظ — قرار أي طرف لا يُحذف.
  • كل زر محاكاة موسوم «(محاكاة)» في نصه — لا ادعاء تكامل غير مؤكد.
  • عربي أولًا RTL؛ الأرقام والبصمات جزر LTR داخل النص.
