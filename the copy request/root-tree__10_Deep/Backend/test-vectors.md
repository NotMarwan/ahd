# 🧪 Test vectors — every computed claim, reproducible

> Every number in [[seal-scheme-spec]], [[muqassa-deep]], and
> [[trust-signal-and-graph-analytics]] is emitted by the reference engine, not hand-typed.
>
> **Reproduce:**
> ```
> cd 10_Deep/Backend
> node ref/generate-vectors.mjs            # human-readable
> node ref/generate-vectors.mjs --json     # machine-readable (writes ref/vectors.json)
> ```
> Hashing: Node `crypto` SHA-256 (FIPS 180-4). The prototype ships a from-scratch SHA-256;
> `selfTestSha256()` pins both to the NIST vectors so they agree bit-for-bit. Both Node and Chrome
> run V8, so `JSON.stringify`/integer formatting (the JCS primitives) are byte-identical across the
> reference engine and the in-browser demo.

## 5. Muqassa — the demo circle (9 IOUs, 5 friends)

Balances (halalas): `نورة −90000 · سارة 0 · خالد +60000 · ليلى 0 · فهد +30000` (Σ = 0 ✓).

| transfer | halalas | SAR |
|---|---|---|
| نورة → خالد | 60000 | 600 |
| نورة → فهد | 30000 | 300 |

- **9 IOUs → 2 transfers.** Conservation: `allZero=true`, `sumZero=true`; Σ paid = Σ received = 90000 halalas.
- Efficiency: `transfer_reduction = 0.7778` (1 − 2/9); `cash_reduction = 0.5000` (1 − 90000/180000);
  directed cycles detected = **2**. Greedy = exact = **2**.

### 5.1 Edge cases

| case | result |
|---|---|
| pure equal 3-cycle (A→B→C→A) | **0** transfers |
| star (A,B,C → D) | **3** transfers (= P−1) |
| owes & owed equally (A↔B 500) | balances `{A:0, B:0}`, both drop out |

### 5.2 Optimality — worst case (greedy > optimal), machine-found & verified

```
balances:  P0=−2  P1=+4  P2=−5  P3=+2  P4=+1     (Σ = 0)
greedy  → 4 transfers:  P2→P1:4  P2→P3:1  P0→P3:1  P0→P4:1     (verifyPlan ✓)
optimal → 3 transfers:  P2→P1:4  P0→P3:2  P2→P4:1               (verifyPlan ✓)
```
Greedy ≤ P−1 always; optimal in the common case; the exact minimum is NP-hard (solver:
`minTransfersExact`).

### 5.3 Multi-currency (net per currency, never cross-net)

```
input:  A→B 30000 SAR, B→A 20000 SAR, A→C 10000 USD
output: SAR → [A→B : 10000]    USD → [A→C : 10000]      (two independent plans)
```

## 7. Where these run

- **Reference engine** (authoritative, browser-independent): `ref/ahd-ref.mjs` + `ref/generate-vectors.mjs`. `node ref/generate-vectors.mjs` reproduces §0–§6.
- **The prototype** `project/ahd-demo/index.html` runs the same SHA-256, integer-halalas money, and deterministic netting on screen. The **RFC-8785 JCS SEAL** and the **computed trust signal** are provided as a ready-to-apply drop-in in [[prototype-compute-patch]] (handed to the file's active owner, Claude-Hardening, to avoid a concurrent-edit clobber — see [[STATUS]] and the coordination log). The current demo already computes a real SHA-256 hash-chain + live tamper-verifier on integer halalas; the patch raises it to JCS depth and makes the trust ring a computed windowed ratio.
- **Cross-engine check:** because Node and Chrome share V8 and the from-scratch SHA-256 is NIST-pinned, applying the patch yields on-screen `terms_hash`/`leaf`/`bank_sig` byte-identical to §2.
